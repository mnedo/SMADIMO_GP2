## Wikipedia Scraping ## 

Начать мы решили с определения границ рассматрвиаемых городов России - для этого было спаршено несколько страниц википедии: 
[Список городов России](https://ru.wikipedia.org/wiki/%D0%A1%D0%BF%D0%B8%D1%81%D0%BE%D0%BA_%D0%B3%D0%BE%D1%80%D0%BE%D0%B4%D0%BE%D0%B2_%D0%A0%D0%BE%D1%81%D1%81%D0%B8%D0%B8),
[Список аэропортов России](https://ru.wikipedia.org/wiki/%D0%A1%D0%BF%D0%B8%D1%81%D0%BE%D0%BA_%D0%B0%D1%8D%D1%80%D0%BE%D0%BF%D0%BE%D1%80%D1%82%D0%BE%D0%B2_%D0%A0%D0%BE%D1%81%D1%81%D0%B8%D0%B8#cite_note-:20-4), а также со страница городов России попереходили на страницы этих городов (например, 
[Анапа](https://ru.wikipedia.org/wiki/%D0%90%D0%BD%D0%B0%D0%BF%D0%B0)
). Итого мы получили следующий датасет, состоящий из 1125 объектов, у 75 городов есть аэоропорты (изначально примэплилось только 68, остальные 7 были выявлены и обработаны вручную) :

| Город | Регион | Федеральный округ | Население | Наличие аэропорта |
| :-: | :-: | :-: | :-: | :-: | 
| номинальный признак | категориальный признак | категориальный признак | числовой признак (int) | бинарный признак |


Далее убрали те регионы, в которых нет аэропорта из-за ограниченности транспортной доступности(из 1125 городов осталось 878). Потом убрали самые маленькие города (высчитали 10% городов с самым большим население внутри каждого федерального округа), в которых нет аэропорта - у нас осталось 103 города в 63 регионах. Этот список мы и будем рассматривать далее. 
## Ostrovok Scraping ## 

Далее мы провели сбор данных об отелях с сайта [Ostrovok](https://ostrovok.ru/) с помощью парсинга. Сбор выполнялся по всем целевым городам с автоматическим обходом всех страниц отелей каждого города. 
Процесс сбора данных состаял из 2-х этапов:
1) Получение списка отелей
   
   Из JSON извлекался список отелей по каждому городу. Отсюда мы получили: ссылки на страницы отелей и координаты отелей.
3) Сбор детальной информации
   
   Далее выполнялся парсинг самих карточек отелей. Для каждого объекта происходил переход на его страницу и извлекалась следующая информация: название отеля, цена за ночь, рейтинг отеля и количество отзывов.

Таким образом, итоговой датасет стал содержать следующую информацию:

city_name - название города,

hotel_name - название отеля,

reviews_count - количество отзывов на отель,

rating_value - рейтинг отеля,

price_per_night - цена за ночь в отеле,

latitude - широта (точное расположение отеля),

longitude - долгота (точное расположение отеля).

| city_name | hotel_name | reviews_count | rating_value | price_per_night | latitude | longitude
| :-: | :-: | :-: | :-: | :-: | :-: | :-: | 
| категориальный номинальный признак | категориальный номинальный признак | количественный дискретный признак | количественный непрерывный признак | количественный дискретный признак | количественный непрерывный признак | количественный непрерывный признак |

В итоге: 
Количество объектов: 27555.
Каждая строка соответствует одному отелю.

В некоторых столбцах присутствуют пропуски в полях reviews_count и rating_value (т.е. у некоторых отелей нет оценок и отзывов). Также цены изначально были представлены в текстовом виде, поэтому были приведены к нужному виду.

Для подготовки данных мы занимались извлечением числовых значений из строк (цена, рейтинг, количество отзывов), удалением лишних символов и приведением типов данных к числовым.
